《心理疲惫度测试》需求文档

1. 项目简介 & 定位

1.1 产品名称

心理疲惫度测试

1.2 产品定位

一个在线自测工具，帮助用户衡量当前的「情绪疲惫程度」，并识别「疲惫来源类型」及「主要掉血场景」，生成一份个性化的分析报告。

强调：自我觉察工具，不是医疗诊断。

1.3 目标用户

长期处于压力环境的职场人、自由职业者、学生等。

自我评价为「很累、心累、提不起劲」，但又说不清原因的人。

想理解「自己为什么总是这么累」和「可以从哪几方面调整」的人。

1.4 使用场景

小红书 / 公众号 / 短视频引流到测评页，用户完成测评并下载报告。

个人自用：阶段性测评当前状态。

2. 技术与整体原则

2.1 技术边界

仅前端实现，不开发后台管理系统。

所有题目、计算逻辑、结果文案模板等均写死在前端配置中（。

不做用户登录、不存储个人数据（除非简单匿名统计，当前版本可以暂不实现）。

2.2 禁止/不需要的功能

不做分享功能：不提供一键分享至社交媒体的按钮。

不做复杂的后台数据可视化。

2.3 必须实现的关键点

引导用户完成 40 道题的测评。

根据答案计算：

整体疲惫等级

场景掉血情况

疲惫来源主类型 + 副标签

恢复力强弱

基于上述结果，通过 AI 分析生成个性化的报告内容。

生成个性化分析报告，并支持一键下载 PDF 报告（前端生成，无需后端）。

4. 题目架构（Question Architecture）
4.1 总体

总计 40 题，以 4 个模块组织：

模块 A：整体疲惫强度（8 题）

模块 B：疲惫场景（12 题：工作/学习、人际/家庭、自我要求）

模块 C：疲惫来源类型（16 题：4 类型 × 各 4 题）

模块 D：恢复资源 / 保护因子（4 题）

5. 计算逻辑（Scoring & Typing Logic）
5.1 输入

answers: Record<string, number>

key：题目 id（如 "A1"）

value：1–5 的整数分数（完全不符合 → 非常符合）

5.2 输出结构建议
interface CalcResult {
  // 整体疲惫强度
  fatigueAvg: number;            // A模块平均分（1-5）
  fatigueLevel: 0 | 1 | 2 | 3 | 4;
  fatigueLabel: string;          // 如「3级 · 高度疲惫」

  // 场景掉血情况（B模块）
  sceneScores: {
    work: number;                // B1-4 平均 1-5
    relationship: number;        // B5-8 平均 1-5
    selfDemand: number;          // B9-12 平均 1-5
  };

  // 疲惫来源类型匹配度（C模块）
  typeMatches: {
    subtype: FatigueSubtype;
    avgScore: number;            // 该类型题目平均分 1-5
    matchScore: number;          // 匹配度 0-100
  }[];
  primaryType: FatigueSubtype;   // 主类型（最高匹配度）
  secondaryType: FatigueSubtype; // 副标签（第二高匹配度）

  // 恢复资源 / 保护因子（D模块）
  recoveryAvg: number;           // 平均 1-5
  recoveryLevel: "low" | "medium" | "high";
  recoveryLabel: string;         // 如「恢复力偏低 · 休息难以真正回血」

  // 副标签（辅助描述文案的tag）
  tags: string[];                // 例如 ["工作/学习耗竭", "习惯扛责难以说“不”", ...]
}

5.3 计算步骤概述

整体疲惫强度（模块 A）

取 A1–A8 的平均分 fatigueAvg（范围 1–5）。

映射为疲惫等级 fatigueLevel：

avg < 2.0    → 0级：安全区
2.0–2.9      → 1级：轻度疲惫
3.0–3.4      → 2级：中度疲惫
3.5–3.9      → 3级：高度疲惫
≥ 4.0        → 4级：极度疲惫


同时生成对应文案 fatigueLabel：

0: "0级 · 安全区"

1: "1级 · 轻度疲惫"

2: "2级 · 中度疲惫"

3: "3级 · 高度疲惫"

4: "4级 · 极度疲惫"

场景掉血情况（模块 B）

按子模块分别求平均分：

work         = avg(B1-B4)    // 工作/学习
relationship = avg(B5-B8)    // 人际/家庭
selfDemand   = avg(B9-B12)   // 自我要求/内心压力


得出 sceneScores，后续用于结果文案 & tag。

疲惫来源类型匹配度（模块 C）

每种类型对应 4 个题目：

emotional_overload   = avg(C1-C4)
responsibility_trap  = avg(C5-C8)
comparison_anxiety   = avg(C9-C12)
high_pressure_mode   = avg(C13-C16)


对每个类型，算出平均分 avgScore（1–5），再映射为匹配度 matchScore（0–100）：

matchScore = round( (avgScore - 1) / 4 * 100 )


将这些类型按 matchScore 从高到低排序：

最⾼的 subtype 作为 primaryType（主类型）

第二高作为 secondaryType（副标签）

恢复资源 / 保护因子（模块 D）

recoveryAvg = avg(D1-D4)，范围 1–5。

映射为恢复力等级：

avg === 0        → "low"    + "恢复力数据不足"
avg < 2.5        → "low"    + "恢复力偏低 · 休息难以真正回血"
2.5 ≤ avg < 3.8  → "medium" + "恢复力中等 · 有一些修复资源，但仍需主动经营"
≥ 3.8            → "high"   + "恢复力较好 · 懂得为自己充电"


tags（副标签）生成规则（示例逻辑）

根据场景分数：

work ≥ 3.2         → "工作/学习耗竭"
relationship ≥ 3.2 → "人际/家庭消耗明显"
selfDemand ≥ 3.2   → "自我要求偏高、自我苛责"


根据恢复力：

recoveryAvg < 2.5 → "恢复力不足、休息效果有限"
2.5–3.8           → "恢复力一般，有提升空间"
≥ 3.8             → "恢复力尚可，有一定自我修复能力"


根据主类型（若匹配度 ≥ 60）：

emotional_overload   → "高敏感、高共情，容易情绪过载"
responsibility_trap  → "习惯扛责，难以说“不”"
comparison_anxiety   → "易受比较刺激，容易对比焦虑"
high_pressure_mode   → "习惯高压运转，不敢轻易停下"


合并为 tags: string[]，用于个性化文案。

6. 输出报告要求
6.0 个性化报告生成要求（非常重要）
用一套「打标签 + 选模板片段 + 按固定顺序拼接」的规则，把前端写死的文字组合成不同的报告。

具体就是：

先用测评分数，把每个用户打上几个标签：疲惫等级、主/副类型、场景高低、恢复力档位、附加特点。

为每一种标签/档位事先写好短小但有力度的模板片段。

按你设计好的报告结构（总评 → 类型 → 场景 → 恢复力 → 建议 → 收尾），用规则挑片段、拼在一起。


6.1 Web 结果页

内容模块：

顶部总结

「你的情绪疲惫等级：X 级 · XXX」（使用 fatigueLabel）

简短说明 1–2 句（前端可用模板 +变量渲染）。

疲惫来源类型画像

主导类型：primaryType，显示类型名称 + 匹配度（示例：82/100）。

次要类型：secondaryType，显示名称 + 匹配度。

每种类型有一段固定模板文案（写死在前端，根据 subtype 选择）：

典型心态描述

常见行为模式

容易掉入的陷阱

场景掉血情况

文本或简单条形图（可选）展示：

工作/学习：sceneScores.work

人际/家庭：sceneScores.relationship

自我要求：sceneScores.selfDemand

对高分场景给出1–2句解释。

恢复力评估

展示 recoveryLabel 文案。

简短说明当前恢复资源状况。

重点洞察小结

基于 fatigueLevel + primaryType + tags，拼接 2–3 条短句，让用户产生「被看见」感受。

示例结构：

你不是懒，而是长期处在【高压模式 + 扛责习惯】下，工作和人际都在同时消耗你。

目前你的恢复力偏弱，哪怕休息时间不算少，也很难真正回血。

行动建议概览（简版）

不需要太长，只给出：

本周可尝试的 1–2 个小行动

提示：若持续高度/极度疲惫，建议考虑专业支持

下载按钮

「下载完整报告（PDF）」→ 触发前端 PDF 生成。

6.2 PDF 报告

技术要求：

前端使用 JS PDF 库（如 jsPDF / pdf-lib / pdfmake 等）在浏览器端生成。

A4 纵向，3–4 页左右。

文本内容不需要动态分页非常精确，可以略有空白，但不要截断严重。

内容结构建议：

封面页

标题：FBI 心理疲惫度测试 · 个人报告


生成日期

免责声明（简单一句话）：

「本报告仅用于个人自我觉察，不构成任何医疗或心理诊断建议。」

概览页（Summary）

整体疲惫等级：使用 fatigueLabel + 简短解释。

场景雷达或简单表格：

工作/学习、人际/家庭、自我要求 三项分值。

疲惫来源类型：

主类型 + 副类型 + 匹配度。

恢复力等级：recoveryLabel。

一小段「核心结论」：100--200字，基于 primaryType + sceneScores + tags。

详细解读页：类型 & 场景

对主类型做详细解释：

类型名称 + 简介

典型心态 & 内心 OS

在生活中常见的具体表现（列举）

该类型容易形成的恶性循环模式

对次要类型做简短解释。

综合场景结果：

哪些场景分数偏高 → 说明这些地方是主要「掉血源」。

连接主类型说明：例如「责任绑架型 + 家庭/人际高分」组合，意味着什么。

行动建议页

分为 3 个层次：

本周可以尝试的小行动（微调建议）

接下来 1–3 个月可以考虑的调整方向（节奏、边界、恢复力）

何种情况建议寻求专业支持（心理咨询/医生/HR 等）

根据 fatigueLevel 调整建议“力度”：

0–1 级：偏预防、微调

2–3 级：需要主动调整与自我照顾

4 级：明显建议寻求外部支持
